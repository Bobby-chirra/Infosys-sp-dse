A. Data Structures Patterns (Java Examples)

1
Two Pointers Pattern

int i = 0, j = n-1;
while(i < j){
  // logic - move i forward or j backward based on condition
  if(condition) i++;
  else j--;
}
Use for: pairs, palindrome check, sorted array search
Complexity: O(n)
Example: Two Sum (sorted array)

2
Sliding Window Pattern

int i = 0, max = 0;
for(int j=0; j<n; j++){
  // expand window using s[j]
  while(conditionBroken){
    // shrink from i
    i++;
  }
  // record max window size
  max = Math.max(max, j - i + 1);
}
Use for: longest substring without repeat, max sum subarray (fixed window)
Complexity: O(n)

3
HashMap Frequency Pattern

Map<Character,Integer> map = new HashMap<>();
map.put(c, map.getOrDefault(c, 0) + 1);
Use for: anagrams, counting, prefix problems
Complexity: O(1) average for operations

4
Linked List Essentials

Reverse Linked List:

ListNode prev = null, curr = head;
while(curr != null){
  ListNode nxt = curr.next;
  curr.next = prev;
  prev = curr;
  curr = nxt;
}
return prev;
Detect cycle: Floyd's (slow/fast pointers)
Complexity: O(n) for traversal

5
Tree Traversals

void inorder(TreeNode root){
  if(root == null) return;
  inorder(root.left);
  print(root.val);
  inorder(root.right);
}
Level Order: Queue<TreeNode> q = new LinkedList<>()
Complexity: O(n) for all traversals

6
Graph BFS Pattern

Queue<Integer> q = new LinkedList<>();
boolean[] vis = new boolean[n];
q.add(src); vis[src] = true;

while(!q.isEmpty()){
  int cur = q.poll();
  for(int nei : adj[cur]){
    if(!vis[nei]){
      vis[nei] = true;
      q.add(nei);
    }
  }
}
Use for: shortest path in unweighted graphs, connected components
Complexity: O(V + E)

B. SQL Quick Patterns

7
JOIN Template

SELECT a.col1, b.col2
FROM tableA a
INNER JOIN tableB b ON a.id = b.a_id;
Types: INNER, LEFT, RIGHT, FULL OUTER JOIN

8
GROUP BY Template

SELECT dept, COUNT(*) AS total
FROM employees
GROUP BY dept
HAVING COUNT(*) > 5;
Remember: WHERE filters rows, HAVING filters groups

C. OOP + Core Java

4 Pillars of OOP
  • Encapsulation: Data hiding with getters/setters
  • Inheritance: extends keyword, IS-A relationship
  • Polymorphism: Method overriding (runtime)
  • Abstraction: Abstract classes & interfaces
Collections Framework
  • List: Ordered, allows duplicates (ArrayList, LinkedList)
  • Set: Unique elements (HashSet, TreeSet)
  • Map: Key-value pairs (HashMap, TreeMap)
Interface Tip: All methods abstract by default (Java ≤7), default methods in Java 8+
